Features

The unglamorous parts, done right

Every DIY Telegram integration hits the same four walls. DifyGram ships past them in about 1,700 lines of typed, tested Worker code.

Streaming edits

The reply grows inside one message: throttled editMessageText every 1.1 s, retry_after honored, cursor removed when the stream closes. Feels like ChatGPT, works over a plain webhook.

Conversation memory

chat_id to conversation_id in Workers KV. One write per new conversation, so the free tier lasts forever. /reset starts clean, and stale ids retry fresh automatically.

Formatting that survives

Markdown becomes Telegram HTML, never MarkdownV2. If Telegram rejects the markup, the Worker resends as plain text instead of failing. Code blocks arrive intact.

Any backend

BACKEND_MODE=generic posts to any HTTP endpoint and understands plain text, {"reply"}, {"output"}, {"text"}, and n8n response arrays. Dify streaming stays the default.

Zero dependencies

The runtime is the platform: fetch, KV, Cache API. No node_modules in production, no framework, no supply chain to audit.

Boring reliability

The failure modes are handled before you meet them.

Setup

Live in five minutes

Three steps and one free Cloudflare account. The deploy button provisions the KV namespace for you.

1

Create the bot

Message @BotFather, send /newbot, copy the token.

2

Deploy the Worker

One click provisions everything, then set three secrets.

Deploy to Cloudflare ↗ 

wrangler secret put TELEGRAM_BOT_TOKEN
wrangler secret put DIFY_API_KEY
wrangler secret put WEBHOOK_SECRET
3

Wire the webhook

Open this URL once, then message your bot.

https://<your-worker>.workers.dev/setup?secret=<WEBHOOK_SECRET>
